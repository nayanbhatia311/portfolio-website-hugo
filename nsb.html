Overview

NSB (Network Simulation Bridge) lets you connect real or emulated applications to network simulators so you can evaluate system behavior under controllable, repeatable network conditions (e.g., latency, loss, bandwidth profiles) without rewriting the app logic.

⚡ Drop-in bridging

Proxy real app traffic into simulations with minimal app changes.

🔧 Deterministic tests

Script repeatable scenarios (mobility, link failures, congestion).

📖 Research-ready

Designed for methodical experimentation and publishable results.

Docs are a developer-oriented draft based on the public repo; fill in project-specific details as the API stabilizes.

Quickstart

Clone the repo and build the bridge and example components.

# 1) Clone
git clone https://github.com/nsb-ucsc/nsb
cd nsb

# 2) (Optional) create a Python venv for any Python utilities
python3 -m venv .venv && source .venv/bin/activate
python -m pip install -U pip wheel

# 3) Build native components (e.g., C++/Make)
make -j$(sysctl -n hw.ncpu || nproc)

# 4) Run a minimal smoke test
./bin/nsb-bridge --help

If you use a simulator (e.g., OMNeT++), install it first and ensure its toolchain is in your PATH.

Install

Prerequisites

  • Build toolchain: recent C++ compiler, make or cmake.
  • Python 3.10+ for helper scripts.
  • (Optional) Supported simulator. See “Simulator Integrations”.

From source

git clone https://github.com/nsb-ucsc/nsb
cd nsb
make           # or cmake -S . -B build && cmake --build build

Core Concepts

  • Bridge Node: connects app endpoint to simulated network.
  • Adapters: modules translating I/O to events.
  • Scenario: describes links, mobility, failures.
  • Determinism: reproducible runs with fixed seeds.

Examples

Below are stubbed examples you can adapt.

Run an echo server

# Terminal A
echo '{"links":[{"src":"app","dst":"net","latency_ms":40,"loss":0.01}]}' > scenario.json
./bin/nsb-bridge --scenario scenario.json --listen 127.0.0.1:9000

# Terminal B
APP_ENDPOINT=127.0.0.1:9000 ./your_app --server

# Terminal C
echo "ping" | nc 127.0.0.1 9000
